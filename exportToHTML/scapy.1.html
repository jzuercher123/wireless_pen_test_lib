3    .SH NAME
4    scapy \- Interactive packet manipulation tool
5    .SH SYNOPSIS
6    .B scapy
7    .RI [ options ]
8    .SH DESCRIPTION
9    This manual page documents briefly the
10   .B Scapy
11   tool.
12   .PP
13   \fBScapy\fP is a powerful interactive packet manipulation tool, 
14   packet generator, network scanner, network discovery, packet sniffer,
15   etc. It can for the moment replace hping, parts of nmap, arpspoof, arp-sk,
16   arping, tcpdump, tshark, p0f, ...
17   .PP
18   \fBScapy\fP uses the Python interpreter as a command board. That means that 
19   you can use directly Python language (assign variables, use loops, 
20   define functions, etc.) If you give a file a parameter when you run 
21   \fBScapy\fP, your session (variables, functions, instances, ...) will be saved 
22   when you leave the interpreter and restored the next time you launch
23   \fBScapy\fP. 
24   .PP
25   The idea is simple. Those kinds of tools do two things : sending packets 
26   and receiving answers. That's what \fBScapy\fP does : you define a set of 
27   packets, it sends them, receives answers, matches requests with answers 
28   and returns a list of packet couples (request, answer) and a list of 
29   unmatched packets. This has the big advantage over tools like nmap or 
30   hping that an answer is not reduced to (open/closed/filtered), but is 
31   the whole packet. 
32   .PP
33   On top of this can be used to build more high-level functions, for example, one 
34   that does traceroutes and give as a result only the start TTL of the 
35   request and the source IP of the answer. One that pings a whole network 
36   and gives the list of machines answering. One that does a portscan and 
37   returns a LaTeX report. 
38   
39   .SH OPTIONS
40   Options for Scapy are:
41   .TP
42   \fB\-h\fR
43   display usage
44   .TP
45   \fB\-H\fR
46   header-less mode, also reduces verbosity.
47   .TP
48   \fB\-d\fR
49   increase log verbosity. Can be used many times.
50   .TP
51   \fB\-s\fR FILE
52   use FILE to save/load session values (variables, functions, instances, ...) 
53   .TP
54   \fB\-p\fR PRESTART_FILE 
55   use PRESTART_FILE instead of $HOME/.config/scapy/prestart.py as pre-startup file
56   .TP
57   \fB\-P\fR
58   do not run prestart file
59   .TP
60   \fB\-c\fR STARTUP_FILE 
61   use STARTUP_FILE instead of $HOME/.config/scapy/startup.py as startup file
62   .TP
63   \fB\-C\fR
64   do not run startup file
65   
66   .SH COMMANDS
67   Only the vital commands to begin are listed here for the moment.
68   .TP
69   \fBls()\fR
70   lists supported protocol layers.
71   If a protocol layer is given as parameter, lists its fields and types of fields.
72   If a string is given as parameter, it is used to filter the layers.
73   .TP
74   \fBlsc()\fR
75   lists scapy's main user commands.
76   .TP
77   \fBconf\fR
78   this object contains the configuration. 
79   
80   .SH FILES
81   \fB$HOME/.config/scapy/prestart.py\fR
82   This file is run before Scapy core is loaded. Only the \fBconf\fP object
83   is available. This file can be used to configure the CLI, configure
84   parameters such as the \fBconf.load_layers\fP  list to choose which layers
85   will be loaded, or change the logging level (for instance):
86   
87   .nf
88   conf.interactive_shell = "bpython"
89   log_loading.setLevel(logging.WARNING)
90   conf.load_layers.remove("bluetooth")
91   conf.load_layers.append("new_layer")
92   .fi
93   
94   \fB$HOME/.config/scapy/startup.py\fR
95   This file is run after Scapy is loaded. It can be used to configure
96   more of Scapy behaviors, like un-registering layers:
97   
98   .nf
99   conf.prog.pdfreader = "xpdf"
100  split_layers(UDP,DNS)
101  .fi

103  .SH EXAMPLES
104  
105  More verbose examples are available in the documentation at
106  \fIhttps://scapy.readthedocs.io/\fP.
107  Just run \fBscapy\fP and try the following commands in the interpreter.
108  
109  .LP
110  Test the robustness of a network stack with invalid packets:
111  .nf
112  sr(IP(dst="172.16.1.1", ihl=2, options=["verb$2"], version=3)/ICMP(), timeout=2)
113  .fi
114  
115  .LP
116  Packet sniffing and dissection (with a bpf filter or tshark-like output):
117  .nf
118  a=sniff(filter="tcp port 110")
119  a=sniff(prn = lambda x: x.show) 
120  .fi
121  
122  .LP
123  Sniffed packet re-emission:
124  .nf
125  a=sniff(filter="tcp port 110")
126  sendp(a)
127  .fi
128  
129  .LP
130  Pcap file packet re-emission:
131  .nf
132  sendp(rdpcap("file.cap"))
133  .fi
134  
135  .LP
136  Manual TCP traceroute:
137  .nf
138  sr(IP(dst="www.google.com", ttl=(1,30))/TCP(seq=RandInt(), sport=RandShort(), dport=dport)
139  .fi
140  
141  .LP
142  Protocol scan:
143  .nf
144  sr(IP(dst="172.16.1.28", proto=(1,254)))
145  .fi
146  
147  .LP
148  ARP ping:
149  .nf
150  srp(Ether(dst="ff:ff:ff:ff:ff:ff")/ARP(pdst="172.16.1.1/24"))
151  .fi
152  
153  .LP
154  ACK scan:
155  .nf
156  sr(IP(dst="172.16.1.28")/TCP(dport=(1,1024), flags="A"))
157  .fi
158  
159  .LP
160  Passive OS fingerprinting:
161  .nf
162  sniff(prn=prnp0f) 
163  .fi
164  
165  .LP
166  Active OS fingerprinting:
167  .nf
168  nmap_fp("172.16.1.232")
169  .fi
170  
171  
172  .LP
173  ARP cache poisoning:
174  .nf
175  sendp(Ether(dst=tmac)/ARP(op="who-has", psrc=victim, pdst=target))
176  .fi
177  
178  .LP
179  Reporting:
180  .nf
181  report_ports("192.168.2.34", (20,30))
182  .fi

192  .SH BUGS
193  Does not give the right source IP for routes that use interface aliases.
194  
195  May miss packets under heavy load. This is a restriction from python itself
196  
197  Session saving is limited by Python ability to marshal objects. As a 
198  consequence, lambda functions and generators can't be saved, which seriously
199  reduce the usefulness of this feature. 
200  
201  BPF filters don't work on Point-to-point interfaces.
